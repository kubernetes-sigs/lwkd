## Subprojects and Dependency Updates

* [**CRI-O v1.33.9**](https://github.com/cri-o/cri-o/releases/tag/v1.33.9) continues patch-level support for the 1.33 release line with no listed functional changes or dependency updates.
* [**cluster-autoscaler v1.35.0**](https://github.com/kubernetes/autoscaler/releases/tag/cluster-autoscaler-1.35.0) introduces major improvements including CapacityBuffer v1beta1 support, quota-aware capacity buffers, new GPU resource support, CSI volume limit-aware scaling, new scale-down performance metrics, and additional configuration flags for node startup time and scheduling parallelism. It also includes multiple bugfixes, updates Kubernetes libraries to v1.35.0, and refreshes cloudprovider integrations across AWS, Azure, GCE, Cluster API, Hetzner, and others.
* [**cluster-autoscaler patch releases (v1.32.6, v1.33.4, v1.34.3)**](https://github.com/kubernetes/autoscaler/releases) provide backported stability fixes and cloudprovider updates across supported release branches; [**v1.34.3**](https://github.com/kubernetes/autoscaler/releases/tag/cluster-autoscaler-1.34.3) adds AWS ignore-label updates, refreshes the EC2 instance list, improves scale-down eligibility handling, expands GPU support, and fixes log stream handling; [**v1.33.4**](https://github.com/kubernetes/autoscaler/releases/tag/cluster-autoscaler-1.33.4) refreshes the EC2 instance list, improves OCI node deletion behavior, fixes scale-down eligibility edge cases, and corrects log stream handling; [**v1.32.6**](https://github.com/kubernetes/autoscaler/releases/tag/cluster-autoscaler-1.32.6) improves release automation, updates default registry settings, refreshes the EC2 instance list, and includes OCI deletion and scale-down eligibility fixes.
