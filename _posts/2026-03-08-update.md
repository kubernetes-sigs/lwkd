## Featured PRs

* [#136976: Create Workload API v1alpha2](https://github.com/kubernetes/kubernetes/pull/136976)

Introduces `scheduling.k8s.io/v1alpha2` Workload and PodGroup APIs as part of ongoing workload-aware scheduling work. This replaces the previous `v1alpha1` Workload API and updates Pods to reference scheduling groups through a new `SchedulingGroup` field, decoupling PodGroups from Workloads and enabling more flexible workload-level scheduling.

## Other Merges

* Fix informer-gen to [generate SetTransform calls](https://github.com/kubernetes/kubernetes/pull/137473) that correctly override per-informer transforms.
* Fix [kubectl describe node](https://github.com/kubernetes/kubernetes/pull/137394) to correctly display resource requests and limits for pods using pod-level resources.
* [`v1alpha1` WebhookAdmissionConfiguration](https://github.com/kubernetes/kubernetes/pull/137379) has been removed. It was deprecated in v1.17 in favor of `apiserver.config.k8s.io/v1`.
* Remove the [dead `--bounding-dirs` flag and `BoundingDirs` field](https://github.com/kubernetes/kubernetes/pull/137348) from deepcopy-gen.
* Fix incorrect behavior when using AllocationModeAll with [DRA PrioritizedList](https://github.com/kubernetes/kubernetes/pull/137347) that prevented the allocator from successfully allocating a claim even when devices were available.
* [Direct access to the `Raw` field of `metav1.FieldsV1`](https://github.com/kubernetes/kubernetes/pull/137304) is deprecated.
* Remove [misleading `SuggestFor` entries from `kubectl wait`](https://github.com/kubernetes/kubernetes/pull/137266) so that it is no longer suggested when users type `kubectl list` or `kubectl ps`. 
* [GetPCIeRootAttributeByPCIBusID](https://github.com/kubernetes/kubernetes/pull/137220) helper now accepts a `fs.ReadLinkFS` optional argument to be filesystem-independent.
* CRI API: A new field is added to the [PullImageResponse message](https://github.com/kubernetes/kubernetes/pull/137217) - `image_id`.
* DRA DeviceTaintRules:  [TimeAdded of the taint](https://github.com/kubernetes/kubernetes/pull/137167) is not only added automatically, it now also gets updated automatically when changing the effect.
* When [`kubectl exec` or `kubectl logs`](https://github.com/kubernetes/kubernetes/pull/136973) are run with a specified container name, and no container with that name is found, `kubectl` now lists the names of containers that would be valid to specify.
* Ensures [single-container pod can restart](https://github.com/kubernetes/kubernetes/pull/136966) quickly with RestartAllContainers action.
* [Preserve the logs of restarted containers](https://github.com/kubernetes/kubernetes/pull/136963) for containers restarted by feature RestartAllContainers.
* Add [placement-based pod group scheduling algorithm](https://github.com/kubernetes/kubernetes/pull/136944) to scheduler. Its use is guarded by the TopologyAwareWorkloadScheduling feature gate.
* Add multiple conditions support to [kubectl wait](https://github.com/kubernetes/kubernetes/pull/136855) command.
* kubelet: if the [`--client-ca-file` is updated](https://github.com/kubernetes/kubernetes/pull/136762) while kubelet is running, the updated root certificates are now correctly used to advertise accepted authorities to TLS clients connecting to the kubelet endpoints. This behavior is guarded by the `ReloadKubeletClientCAFile` feature gate, which is enabled by default.
* Add the [`timezone` field](https://github.com/kubernetes/kubernetes/pull/136663) to the cronjob describe output.
* Add [`--detach-keys` flag to `kubectl attach` and `kubectl run`](https://github.com/kubernetes/kubernetes/pull/134997), allowing detach without terminating the container.

## Subprojects and Dependency Updates

* [**CoreDNS v1.14.2**](https://github.com/coredns/coredns/releases/tag/v1.14.2): introduces the new `proxyproto` plugin to support Proxy Protocol and preserve client IPs behind load balancers, improves DNS logging metadata, strengthens randomness for loop detection, and fixes issues including TLS+IPv6 forwarding, CNAME rewriting behavior, ACL bypass prevention, and a Kubernetes plugin crash. The release also updates the build to Go 1.26.1 with multiple upstream security fixes.
* [**ingress-nginx releases**](https://github.com/kubernetes/ingress-nginx/releases): [**controller v1.15.0**](https://github.com/kubernetes/ingress-nginx/releases/tag/controller-v1.15.0) adds improvements to template safety, annotation validation, SSL passthrough handling, and controller stability while updating dependencies and CI tooling; also [**controller v1.14.4**](https://github.com/kubernetes/ingress-nginx/releases/tag/controller-v1.14.4) provides backported fixes and dependency updates for the 1.14 branch. Corresponding Helm chart updates include [**helm-chart 4.15.0**](https://github.com/kubernetes/ingress-nginx/releases/tag/helm-chart-4.15.0), [**4.14.4**](https://github.com/kubernetes/ingress-nginx/releases/tag/helm-chart-4.14.4), and [**4.13.8**](https://github.com/kubernetes/ingress-nginx/releases/tag/helm-chart-4.13.8), aligning chart packaging with their respective controller versions.
* [**cloud-provider-alibaba-cloud v2.13.0**](https://github.com/kubernetes/cloud-provider-alibaba-cloud/releases/tag/v2.13.0): adds support for extended TLS certificates on CLB and TCPSSL listeners, introduces cross-AZ forwarding for NLB by default, and improves ENI attachment handling to prevent failures affecting other pods. It also refines internal CLB creation logic by automatically selecting an available vSwitch when none is specified.
* [**containerd v2.2.2**](https://github.com/containerd/containerd/releases/tag/v2.2.2): delivers a patch release with CRI improvements such as better handling of legacy registry mirror configurations, fixes for encrypted image pulls and CNI cleanup after restarts, improved runtime behavior for bind mounts and AppArmor compatibility, and multiple stability fixes including error sanitization and memory metrics handling. The release also includes dependency updates across OpenTelemetry, OAuth2, and gRPC libraries.
