## Other Merges

* Fixed [kubelet failure starting on ZFS](https://github.com/kubernetes/kubernetes/pull/138587) due to missing cadvisor plugin.
* Fix [regression in kubectl resource printing](https://github.com/kubernetes/kubernetes/pull/138550) on bigger data sets (100+ rows).
* Fixed a bug where [Pod `.status.resourceClaimStatuses` could flap](https://github.com/kubernetes/kubernetes/pull/138408) between partial lists of claims when multiple claims were used in the pod.
* kubeadm: [etcd cluster status checks now use a quorum approach](https://github.com/kubernetes/kubernetes/pull/138403) instead of requiring all members to be healthy, so the check won't fail if there are sufficient healthy voting members.
* kubeadm: [fix MemberPromote to skip the etcd promote API call](https://github.com/kubernetes/kubernetes/pull/138390) when the member is already a voting member, avoiding unnecessary retries and timeout.
* kubeadm: [preflight port checks now bind to the configured component address](https://github.com/kubernetes/kubernetes/pull/138250) (via `localAPIEndpoint.address` or `--bind-address` extraArgs) instead of all interfaces, for kube-apiserver, kube-scheduler, kube-controller-manager, and etcd.
* HorizontalPodAutoscaler resources now [generate `metadata.generation` and `status.observedGeneration` fields](https://github.com/kubernetes/kubernetes/pull/138228).
* [Improved error reporting when invoking `kubectl exec`](https://github.com/kubernetes/kubernetes/pull/138214).
* kube-apiserver now [validates the `--advertise-address` IP](https://github.com/kubernetes/kubernetes/pull/138102) when using `--endpoint-reconciler-type` `master-count` or `lease`, ensuring the IP can be persisted to an `Endpoints` API object.
* Fixed [duplicated mount arguments in log string output](https://github.com/kubernetes/kubernetes/pull/138098) from `MakeMountArgsSensitiveWithMountFlags`.
* kubeadm: added the [`kubeproxydaemonset` patch target](https://github.com/kubernetes/kubernetes/pull/138090) to allow patching the kube-proxy DaemonSet during `kubeadm init` and `kubeadm upgrade`, consistent with the existing `corednsdeployment` patch target.
* Kubelet now [enforces explicit HTTP method restrictions for logs-related endpoints](https://github.com/kubernetes/kubernetes/pull/138088): read-only server endpoints reject non-GET methods with 405, and NodeLogQuery allows only GET and POST.
* Deprecated [`MultiLock`, `UnknownLeader`, and `ConcatRawRecord`](https://github.com/kubernetes/kubernetes/pull/138070) in the client-go leader election `resourcelock` package.
* The eviction API now [includes structured `CauseType` values in PodDisruptionBudget-related `Forbidden` errors](https://github.com/kubernetes/kubernetes/pull/138003), so clients can distinguish PDB invalid-state errors without string-matching on the message.
* `kubectl get crd` now [displays additional columns — GROUP, SCOPE, VERSIONS, and CREATED AT](https://github.com/kubernetes/kubernetes/pull/131599) — providing at-a-glance visibility into each CRD's API group, scope, served versions, and creation timestamp.
* Fixed [`kubectl get storageclass` to show only the effective default StorageClass as `(default)`](https://github.com/kubernetes/kubernetes/pull/135964) when multiple StorageClasses have the default annotation.
* Image volume validation now [rejects empty `image.reference` fields in Pod templates](https://github.com/kubernetes/kubernetes/pull/135989) across Deployment, StatefulSet, DaemonSet, Job, and similar resources.
* Fixed [CronJob controller failing to adopt existing Jobs](https://github.com/kubernetes/kubernetes/pull/136920) due to incorrectly using the empty namespace from the JobTemplate.
* Fixed a v1.35 regression where [StatefulSets with Parallel pod management incorrectly counted unavailable pods from older revisions against the `maxUnavailable` budget](https://github.com/kubernetes/kubernetes/pull/137666).
* Added three Pod cluster event subtypes to the scheduler framework — [`AssignedPod`, `UnscheduledPod`, and `TargetPod`](https://github.com/kubernetes/kubernetes/pull/135905) — allowing plugins to register only for the specific pod events they need, improving performance.
* Removed the GA feature gate [`AnyVolumeDataSource`](https://github.com/kubernetes/kubernetes/pull/135336), locked and enabled since v1.33.
* Fixed [stale remote HNS endpoint cleanup on Windows](https://github.com/kubernetes/kubernetes/pull/138000) when a pod IP is reused across nodes in L2Bridge networks, preventing DNS timeouts from traffic being routed to the wrong node.
* Removed the [`--concurrent-service-syncs` kube-controller-manager flag](https://github.com/kubernetes/kubernetes/pull/138002), which has been a no-op since v1.31.
* [Removed `KubeletMinVersion` gate from the DRA multiple ResourceClaims e2e test](https://github.com/kubernetes/kubernetes/pull/138001), as the feature is now sufficiently available.
