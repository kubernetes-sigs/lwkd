## Other Merges

* Fixes the total [pod resources computation](https://github.com/kubernetes/kubernetes/pull/137683)
* Explicitly [writes memory.min=0 for QoS cgroups](https://github.com/kubernetes/kubernetes/pull/137637) when the calculated requests are zero.
* Truncates the [watch cache RV metric](https://github.com/kubernetes/kubernetes/pull/137615) to 15 digits to ensure precision.
* Improve stability by sorting containers by create time and ID in [kubeGenericRuntimeManager.GetPods() and GetPod()](https://github.com/kubernetes/kubernetes/pull/137566)
* Allow the CRI (and NRI) to [block pod-level resizes](https://github.com/kubernetes/kubernetes/pull/137555).
* Slow requests that use impersonation can now be tracked via the [`apiserver.latency.k8s.io/impersonation`](https://github.com/kubernetes/kubernetes/pull/137523) audit event annotation when the ConstrainedImpersonation feature is enabled.
* Enables [Prometheus native histogram support](https://github.com/kubernetes/kubernetes/pull/137466) in scheduler when feature gate is enabled.
* [Fix goroutine hot-loop in client-go](https://github.com/kubernetes/kubernetes/pull/137398) StartEventWatcher when the event broadcaster shuts down before the cancellation context fires.
* With DRABindingConditions enabled, [reusing the same claim across different Pods](https://github.com/kubernetes/kubernetes/pull/137371) could rarely trigger a scheduler panic during parallel de-allocation due to a race condition.
* Add [alpha support for manifest-based admission control](https://github.com/kubernetes/kubernetes/pull/137346) configuration (KEP-5793). 
* Allow users to opt-in to [scheduling behaviour for CSI volume](https://github.com/kubernetes/kubernetes/pull/137343)
* Fix link file ownership of [projected serviceAccountToken](https://github.com/kubernetes/kubernetes/pull/137332).
* Validation messages for a Pod's `status.resourceClaimStatuses[].resourceClaimName` now refer correctly to the [`resourceClaimName` field instead of the `name`](https://github.com/kubernetes/kubernetes/pull/137321) field.
* [Update `kubectl kuberc set`](https://github.com/kubernetes/kubernetes/pull/137300) with options for setting `credentialPluginPolicy` and `credentialPluginAllowlist`.
* Introduces new [staging modules `k8s.io/streaming` and `k8s.io/cri-streaming`](https://github.com/kubernetes/kubernetes/pull/137298) for Kubernetes streaming transport and CRI streaming server code.
* Added two scheduler metrics for Device Binding Conditions, covering [allocation attempts and PreBind duration](https://github.com/kubernetes/kubernetes/pull/137284) with status and driver labels.
* Added [PlacementScore extension point](https://github.com/kubernetes/kubernetes/pull/137201) to the scheduler.
* For performance reasons, [`kubectl describe`](https://github.com/kubernetes/kubernetes/pull/137145) now defaults to showing related events only when describing a single object.
* Add [--tls-curve-preferences flag](https://github.com/kubernetes/kubernetes/pull/137115) for configuring TLS key exchange mechanism.
* Introduce [scheduling.k8s.io/v1alpha2 Workload and PodGroup](https://github.com/kubernetes/kubernetes/pull/136976) API.
* Reflecting the expected replica count to the output of [kubectl scale command](https://github.com/kubernetes/kubernetes/pull/136945).
* [Garbage collector](https://github.com/kubernetes/kubernetes/pull/136817) now correctly handles objects deleted externally, preventing spurious error logs.
* [Add tlsServerName field to EgressSelectorConfiguration TLSConfig](https://github.com/kubernetes/kubernetes/pull/136640) to allow overriding the server name used for TLS certificate verification.
* Add [ControllerManagerReleaseLeaderElectionLockOnCancel feature gate to gate leader election](https://github.com/kubernetes/kubernetes/pull/136279) lock release on exit for kube-controller-manager.
* Extend [WebSocket Streaming Protocol](https://github.com/kubernetes/kubernetes/pull/136256) to the Kubelet for Exec/Attach/PortForward.
* Remove [CRD stored versions](https://github.com/kubernetes/kubernetes/pull/135297) from status upon SVM migration.
* Reduced get [PV request from KCM pv-controller](https://github.com/kubernetes/kubernetes/pull/134290) for CSI volumes.
* k8s.io/client-go/transport now [automatically reloads certificate](https://github.com/kubernetes/kubernetes/pull/132922) authority roots from disk when they are supplied via a file path.
* Allow the [Topology, CPU, and Memory managers](https://github.com/kubernetes/kubernetes/pull/134768) to recognize and act upon `pod.spec.resources`, enabling two flexible resource management models
