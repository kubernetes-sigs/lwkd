## Other Merges

* Use stable curl download for [windows busybox testing image](https://github.com/kubernetes/kubernetes/pull/138879)
* Switch [`StorageVersionMigration`](https://github.com/kubernetes/kubernetes/pull/138874) to use merge patch over SSA
* Add retrying for [Bind API calls](https://github.com/kubernetes/kubernetes/pull/138855)
* kubeadm: kubeadm join now returns a clear error message when the [TLS bootstrap kubeconfig](https://github.com/kubernetes/kubernetes/pull/138853) has a current-context that does not appear in the contexts list, instead of panicking with a nil pointer dereference.
* kubeadm: when fetching cluster-info over [HTTPS during discovery](https://github.com/kubernetes/kubernetes/pull/138852), the HTTP response status code is now checked, so a non-200 response produces a clear error instead of a confusing kubeconfig parse failure.
* Functions and structs that take in [`authorizer.Authorizer`](https://github.com/kubernetes/kubernetes/pull/138801) might now choose to accept only a smaller interface, `authorizer.UnconditionalAuthorizer`, in case only the receiver only needs to perform unconditional authorization requests and wants to signal this in the code for clarity. Any authorizer implementation must still implement the full `authorizer.Authorizer interface`.
* [Workload-aware preemption](https://github.com/kubernetes/kubernetes/pull/138757) now preempts victims so that as many as possible of the preemptor pods can be scheduled.
* Fix the inconsistency between opportunistic batching and PodGroups that made the batching hints always infeasible during [PodGroup scheduling cycle](https://github.com/kubernetes/kubernetes/pull/138754/changes).
* Fix nil pointer dereference in [Windows memory eviction threshold notifier](https://github.com/kubernetes/kubernetes/pull/138727) when `GetPerformanceInfo()` fails.
* Added an alpha feature gate, `[ConsistentListFromCacheSkipTimeoutFallback`](https://github.com/kubernetes/kubernetes/pull/138701/changes) .When enabled, kube-apiserver returns HTTP 429 for consistent LIST requests that cannot be served from watch cache within the timeout window, instead of falling back to storage.
* Ensure [stale cache does not impact the marking of nodes as unhealthy](https://github.com/kubernetes/kubernetes/pull/138698/changes)by checking with a live get.
* kubectl now sets its path in the [`KUBECTL_PATH` environment variable](https://github.com/kubernetes/kubernetes/pull/138694) when executing a plugin.
* kubeadm: skip [LocalAPIEndpoint defaulting on 'kubeadm join'](https://github.com/kubernetes/kubernetes/pull/138692) for worker nodes.
* [kubelet no longer generates an event once per second](https://github.com/kubernetes/kubernetes/pull/138655) for every image volume in a pod.
* client-go will [request `v2` for aggregated discovery and not fall back to `v2beta1`](https://github.com/kubernetes/kubernetes/pull/138271)
* [HPA controller](https://github.com/kubernetes/kubernetes/pull/138294) now reconciles newly created and spec-changed HPAs immediately instead of waiting for the full resync period (default 15s).
* kubelet: [enforce ephemeral-storage limits](https://github.com/kubernetes/kubernetes/pull/138462) on restartable init containers.
* [Metrics are now decremented for pods removed from the active or backoff queues](https://github.com/kubernetes/kubernetes/pull/138482) before scheduling.
* [kubelet no longer fails to start on ZFS](https://github.com/kubernetes/kubernetes/pull/138590) due to a missing cadvisor plugin.
* [`kubectl drain --disable-eviction --dry-run=server`](https://github.com/kubernetes/kubernetes/pull/137543) no longer hangs indefinitely.
* StatefulSet with [`OnDelete` update strategy now correctly updates `Status.CurrentRevision`](https://github.com/kubernetes/kubernetes/pull/136833) after all pods are recreated with the new revision.
