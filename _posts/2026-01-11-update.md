## Other Merges

* [kubeadm](https://github.com/kubernetes/kubernetes/pull/136130) switches tests to `NewClientset`
* The [`endpoints` field](https://github.com/kubernetes/kubernetes/pull/136111) in `discovery.k8s.io/v1` `EndpointSlice` is now optional
* Add [`auditID`](https://github.com/kubernetes/kubernetes/pull/136084) to watch start logs
* Add [logging](https://github.com/kubernetes/kubernetes/pull/136085) when WatchList requests complete their initial listing phase
* [Scheduler](https://github.com/kubernetes/kubernetes/pull/135955) aligns victim metrics semantics between async and sync preemption
* [`csiDriverClient.nodeGetCapabilities`](https://github.com/kubernetes/kubernetes/pull/135930) now returns `NewUncertainProgressError`
* Wire the discovery API and networking API group for declarative validation and migrate [`Endpoint.Addresses` + `EndpointSlice.AddressType`](https://github.com/kubernetes/kubernetes/pull/135890) and [`IngressClassParametersReference.Name`](https://github.com/kubernetes/kubernetes/pull/134909)

* The [ipvs and winkernel](https://github.com/kubernetes/kubernetes/pull/135631) kube-proxy backends are reverted to their pre-1.34 behavior
* Sort the runtime handlers list returned by the [CRI runtime](https://github.com/kubernetes/kubernetes/pull/135358)
* Add [`--concurrent-resourceclaim-syncs`](https://github.com/kubernetes/kubernetes/pull/134701) to configure kube-controller-manager ResourceClaim reconcile concurrency
* Optimize the [DRA allocator](https://github.com/kubernetes/kubernetes/pull/133678)
