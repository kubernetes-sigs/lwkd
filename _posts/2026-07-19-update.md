## Featured PRs

### [140085: Enable EtcdRangeStream feature gate by default](https://github.com/kubernetes/kubernetes/pull/140085)

In this pull request, [Jefftree](https://github.com/Jefftree) enabled the **EtcdRangeStream** feature gate by default, advancing the feature to **Beta** as part of [KEP-5966: Etcd Range Streaming](https://github.com/kubernetes/enhancements/issues/5966). Instead of retrieving entire response sets from etcd in a single, memory-intensive operation, the `kube-apiserver` can now stream large LIST responses incrementally in smaller chunks. This significantly reduces peak memory usage during large list operations and watch cache initialization while improving performance. Benchmark results on 5,000-node clusters show approximately a **1.4× improvement** in watch cache initialization time, making this an important scalability enhancement for large production Kubernetes clusters. See also the related implementation work in the ongoing Watch/List streaming effort, including PRs such as [#139692](https://github.com/kubernetes/kubernetes/pull/139692).

### [139437: ClusterTrustBundles reaches General Availability](https://github.com/kubernetes/kubernetes/pull/139437)

In this pull request, [stlaz](https://github.com/stlaz) promoted **ClusterTrustBundles** and **ClusterTrustBundleProjection** to **General Availability (GA)**, completing the work tracked by [KEP-3257](https://github.com/kubernetes/enhancements/issues/3257). The stable `certificates.k8s.io/v1` API enables cluster administrators to distribute cluster-wide X.509 trust anchors (CA certificates) to workloads through projected volumes, providing a standardized and production-ready mechanism for managing trust bundles. This eliminates the need for custom ConfigMaps or out-of-band certificate distribution methods, simplifying certificate management while improving security and portability for workloads that depend on dynamically managed trust anchors.

## KEP of the Week

### [KEP-5972: SIG Node: Dynamic Containers](https://github.com/kubernetes/enhancements/pull/6169)

KEP-5972, authored by Tim Allclair (@tallclair) in June 2026, proposes allowing main containers to be added to and removed from running Pods, making containers mutable after Pod creation. It is intended to support modern AI and agentic workloads, including high-performance batch jobs and low-latency AI agents, but would also change Kubernetes' long-standing assumption that Pods are immutable execution envelopes.

If adopted, it would require updates to existing ecosystem components such as third-party controllers, service meshes, and logging tools that assume `.spec.containers`  never changes after Pod creation. It also introduces two new Pod subresources: `/dynamic` for adding and removing containers in running Pods, and `/allocated`, which returns the allocated version of the Pod directly from the Kubelet. The KEP is currently undergoing broader design review, with discussion focused on its architectural scope and the impact such a fundamental change could have across the Kubernetes ecosystem.

As this proposal changes one of Kubernetes' core assumptions about Pod immutability, it remains under active discussion for a future Kubernetes release. Follow the ongoing discussion in the [KEP pull request](https://github.com/kubernetes/enhancements/pull/6169) or the [SIG Architecture mailing list](https://groups.google.com/a/kubernetes.io/g/sig-architecture/c/dpsLnuc5Re8).

* Fixes a [DRA bug where `CapacityRequestPolicyRange` did not support fractional quantities](https://github.com/kubernetes/kubernetes/pull/140161), aligning capacity-range behaviour with `resource.Quantity` semantics.
* [Fixes the kube-proxy `--metrics-bind-address` flag documentation](https://github.com/kubernetes/kubernetes/pull/138940), which incorrectly described the default binding behaviour.
* Adds [PodGroup-level scheduling metrics](https://github.com/kubernetes/kubernetes/pull/139840) so operators can observe pod-group counts in scheduling queues, complementing the existing per-pod metrics.
* Introduces the [`Recreate` update strategy for StatefulSets](https://github.com/kubernetes/kubernetes/pull/137187), giving operators a StatefulSet analog to the Deployment `Recreate` strategy.
* apiserver: [`MutatingAdmissionPolicy` and `MutatingAdmissionPolicyBinding` (GA in 1.36) are now stored as v1](https://github.com/kubernetes/kubernetes/pull/137375), fixing autocalculation of the emulated storage version.
* [The `EtcdRangeStream` feature gate is enabled by default (Beta)](https://github.com/kubernetes/kubernetes/pull/140085); watch-cache initialization is ~1.4× faster in unit benchmarks.
* Fixes a [kube-apiserver deadlock on SIGTERM](https://github.com/kubernetes/kubernetes/pull/140241) when its identity Lease cannot be created (e.g. hostname longer than 63 bytes), a regression since v1.35.
* Fixes a [shared `resource.Quantity` mutation bug in DRA counter caches](https://github.com/kubernetes/kubernetes/pull/140518) that could corrupt device accounting.
* Fixes [`DistinctAttribute` handling for DRA requests with multiple devices](https://github.com/kubernetes/kubernetes/pull/140600) in the consumable-capacity path.
* Adds [workload-aware preemption metrics](https://github.com/kubernetes/kubernetes/pull/139373) covering the new preemption paths introduced by KEP-5710.
* [PodLevelResourceManagers metrics gain a stability level](https://github.com/kubernetes/kubernetes/pull/138770) and allocation is now skipped for zero-memory containers.
* Fixes an [ordering bug in the API definitions test suite](https://github.com/kubernetes/kubernetes/pull/140654) that let incorrect `GetResetFields` implementations slip through undetected.
* kubelet: [pod resizes are now retried when resources are shrunk](https://github.com/kubernetes/kubernetes/pull/140646), fixing a case where the retry path was skipped.
* DRA: adds the [`resource.kubernetes.io/numaNode` standard device attribute](https://github.com/kubernetes/kubernetes/pull/139929) (KEP-6072), enabling cross-driver NUMA co-placement via `matchAttribute`.
* Fixes a [nil pointer dereference in the storage version migrator](https://github.com/kubernetes/kubernetes/pull/140586) when a target resource is not in the RESTMapper.
* Implements the [core CompositePodGroup API](https://github.com/kubernetes/kubernetes/pull/139596) under KEP-6012.
* [Container status now accepts uint32 Linux UIDs](https://github.com/kubernetes/kubernetes/pull/138574), removing an incorrect signed-integer validator applied to unsigned UIDs.
* kubectl: [improves error messages when users pass invalid values to `--restart` or `--image-pull-policy`](https://github.com/kubernetes/kubernetes/pull/138188) (e.g. `--restart=never` instead of `Never`).
* Fixes a [CEL reflective wrapper bug](https://github.com/kubernetes/kubernetes/pull/140293) so list concatenation keeps set-list union semantics rather than mutating the LHS.
* The scheduler [flushes pending PodGroup members after others have been assumed](https://github.com/kubernetes/kubernetes/pull/140478), fixing a scheduling flake in gang-scheduling flows.
* Fixes an [overestimation of the pod's resource footprint for multi-container pods undergoing a resize](https://github.com/kubernetes/kubernetes/pull/140047), aggregating desired/allocated/actuated resources per container.
* Fixes a [DRA bug where the structured allocator left reserved state behind](https://github.com/kubernetes/kubernetes/pull/140431) after abandoning a candidate device.
* Fixes [handling of empty pod-level resources](https://github.com/kubernetes/kubernetes/pull/137150) in the PodLevelResources feature.
* [Uses NNN from pod group preemption for all pods in a group](https://github.com/kubernetes/kubernetes/pull/140590), rather than only the suggested pod.

* Fixes a [bug in CEL where `quantity.Add` would mutate the receiver](https://github.com/kubernetes/kubernetes/pull/140556).
* Implements [exclusion of virtual resources from admission webhooks](https://github.com/kubernetes/kubernetes/pull/140019).
