## KEP of the Week

### [KEP-5972: SIG Node: Dynamic Containers](https://github.com/kubernetes/enhancements/pull/6169)

KEP-5972 proposes allowing main containers to be added to and removed from running Pods, making containers mutable after Pod creation. Introduced by SIG Node in June 2026, it would change Kubernetes' long-standing assumption that Pods are immutable execution envelopes. The proposal is intended to support modern AI workloads, including high-performance batch jobs and low-latency AI agents.

If adopted, it would require updates to existing ecosystem components such as third-party controllers, service meshes, and logging tools that assume `.spec.containers` never changes after Pod creation. The proposal also introduces two new Pod subresources: `/dynamic` for adding and removing containers in running Pods, and `/allocated`, which returns the allocated version of the Pod directly from the Kubelet.

As this proposal changes one of Kubernetes' core assumptions about Pod immutability, it is one of the most significant proposals currently under discussion for Kubernetes 1.38.
