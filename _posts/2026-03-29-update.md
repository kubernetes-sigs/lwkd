## Other Merges

* Switch [PLEGOnDemandRelist](https://github.com/kubernetes/kubernetes/pull/137909) default to false for 1.36
* Fixes a 1.35 regression in StatefulSet Parallel pod management by [disabling the MaxUnavailableStatefulSet feature by default](https://github.com/kubernetes/kubernetes/pull/137904).
* Added alpha support for tracking PVC unused status via a [new `Unused` condition on PersistentVolumeClaimStatus](https://github.com/kubernetes/kubernetes/pull/137862).
* Fix bug where users [can't update HPAv2 resources](https://github.com/kubernetes/kubernetes/pull/137856) that use object metrics with averageValue via the v1 HPA API.
* [cri-client helper method NewLogOptions was removed](https://github.com/kubernetes/kubernetes/pull/137827) and LogOptions must be constructed directly.
* Enables Prometheus native histogram support in [kube-proxy](https://github.com/kubernetes/kubernetes/pull/137781),[kubelet](https://github.com/kubernetes/kubernetes/pull/137780), [KCM](https://github.com/kubernetes/kubernetes/pull/137779) when feature gate is enabled.
* The ["Failed to update lease optimistically"](https://github.com/kubernetes/kubernetes/pull/137753) log message may not be shown to users anymore, depending on the log level they have set.
* Add [tiered cgroup v2 memory protection](https://github.com/kubernetes/kubernetes/pull/137719) for `MemoryQoS: memory.min` for Guaranteed pods, `memory.low` for Burstable pods, with node-level metrics and rollback reconciliation (KEP-2570).
* Writes to the [ServiceCIDR main resource](https://github.com/kubernetes/kubernetes/pull/137715) now ignore status field changes in the request, consistent with all other Kubernetes APIs.
* Fixed a bug preventing [Pods sharing ResourceClaims](https://github.com/kubernetes/kubernetes/pull/137647) from being scheduled with GangScheduling.
* Added a deletion protection mechanism for PodGroup objects [KEP-5832](https://github.com/kubernetes/kubernetes/pull/137641).
* Fixed how image names are compared to the values from [`preloadedImagesVerificationAllowlist` in Kubelet's configuration](https://github.com/kubernetes/kubernetes/pull/137629).
* kube-scheduler now updates [PodGroup status with a `PodGroupScheduled`](https://github.com/kubernetes/kubernetes/pull/137611) condition reflecting whether the group was successfully scheduled or is unschedulable.
* [DRA allocation timeouts are now surfaced](https://github.com/kubernetes/kubernetes/pull/137607) as visible scheduling failures and retried with per-pod exponential backoff, instead of silently skipping nodes or incorrectly marking pods unschedulable.
* When [WorkloadAwarePreemption Feature Gate](https://github.com/kubernetes/kubernetes/pull/137606) is enabled, and the Pod Group scheduling fails to find a place for the Pod Group, instead of running default preemption for each pod from the pod group, the workload aware preemption will be run for the whole group.
* [MemoryReservationPolicy cgroup v2 MemoryQoS](https://github.com/kubernetes/kubernetes/pull/137584) support to KubeletConfiguration for memory.min protection (KEP-2570). 
* TAS logic to the pod group scheduling cycle behind [TopologyAwareWorkloadScheduling feature gate](https://github.com/kubernetes/kubernetes/pull/137489). This feature supports scheduling pod groups on nodes with matching topology domains (KEP 5723).
* [`PodGroupPodsCount` scheduler plugin](https://github.com/kubernetes/kubernetes/pull/137488) to support workload-aware scheduling by prioritizing placements with higher pod counts within a group. 
*  [admission plugin](https://github.com/kubernetes/kubernetes/pull/137464) that validates PodGroup resources reference an existing Workload and match the declared PodGroupTemplate spec.
* Disallow setting a resize restart [policy of `RestartContainer` on non-sidecar initContainers](https://github.com/kubernetes/kubernetes/pull/137458), as the resize of such containers has never been supported.
* DRA ResourceSlice controller: new optional [`ReconcilePoolWithName`](https://github.com/kubernetes/kubernetes/pull/137365) allows per-pool reconciliation without setting NodeName on slices.
* Support [in-place pod resize](https://github.com/kubernetes/kubernetes/pull/137352) of running non-sidecar initContainers.
* [Fixed an issue where zero-valued PSI (Pressure Stall Information) metrics](https://github.com/kubernetes/kubernetes/pull/137326) were emitted by the kubelet when the OS does not support PSI, even if the KubeletPSI feature gate was enabled. 
* Warnings and deprecation for [Service.spec.externalIPs](https://github.com/kubernetes/kubernetes/pull/137293)
* [SchedulingConstraints to express TAS constraints](https://github.com/kubernetes/kubernetes/pull/137271) for pod group scheduling behind TopologyAwareWorkloadScheduling feature gate (KEP-5732).
* [Alpha Introduce List Types](https://github.com/kubernetes/kubernetes/pull/137190) for Attributes in DRA (KEP-5491).
* Fixed a bug where [regular containers in a pod with a sidecar](https://github.com/kubernetes/kubernetes/pull/137146) (initContainer with restartPolicy: Always) and a startupProbe failed to restart after crashing following a kubelet restart, leaving affected pods stuck with RestartCount: 0 indefinitely. 
* Opt-in alpha support in [kubeletplugin framework for DRA drivers](https://github.com/kubernetes/kubernetes/pull/137086) to publish DRA Device metadata in pod CDI mounts.
* Integrate [Workload and PodGroup APIs](https://github.com/kubernetes/kubernetes/pull/137032) with the Job controllers to support gang-scheduling.
* ResourcePoolStatusRequest API (v1alpha1) for [querying DRA resource pool availability](https://github.com/kubernetes/kubernetes/pull/137028).
* [Capped `nf_conntrack_max` to 1,048,576](https://github.com/kubernetes/kubernetes/pull/137002) to prevent excessive memory consumption on high-core machines when using automatic calculation.
* [Fixed redundant SSH command executions](https://github.com/kubernetes/kubernetes/pull/137001) in the etcd failure e2e test.
* DRA: PodGroup resources can now make requests with [ResourceClaims through a `spec.resourceClaims` field](https://github.com/kubernetes/kubernetes/pull/136989) which can refer to ResourceClaims and ResourceClaimTemplates. 
* New [RuntimeService streaming RPCs](https://github.com/kubernetes/kubernetes/pull/136987) (`StreamPodSandboxes`, `StreamContainers`, `StreamContainerStats`, `StreamPodSandboxStats`, `StreamPodSandboxMetrics`) and New ImageService streaming RPC (`StreamImages`).
* [Extended NodeResourcesFit](https://github.com/kubernetes/kubernetes/pull/136652) to implement the PlacementScore extension point.
* Introduce [index-based naming in ResourceSlice controller](https://github.com/kubernetes/kubernetes/pull/136641) and ensure ResourceSlices and pools are sorted lexicographically before allocation, allowing users to control allocation priority.
* [`DisruptionMode`, `PriorityClassName` and `Priority` fields](https://github.com/kubernetes/kubernetes/pull/136589) to Workload and PodGroup APIs to support workload-aware preemption when `WorkloadAwarePreemption` feature gate is enabled.
* [Renamed metric](https://github.com/kubernetes/kubernetes/pull/136483) 'etcd_bookmark_counts' to 'etcd_bookmark_total'.
* Introduce [support of DRA](https://github.com/kubernetes/kubernetes/pull/136725) for Native Resources.
* An admission plugin to perform the [OS and node capacity checks](https://github.com/kubernetes/kubernetes/pull/136043)
* The [`UserNamespacesHostNetwork` runtime handler](https://github.com/kubernetes/kubernetes/pull/135828) and integrates the `UserNamespacesHostNetworkSupport` feature gate with the `NodeDeclaredFeatures` feature gate.
* DRA (Dynamic Resource Allocation) drivers and controllers now require [granular RBAC permissions](https://github.com/kubernetes/kubernetes/pull/134947) to update ResourceClaim statuses when the `DRAResourceClaimGranularStatusAuthorization` feature gate is enabled (Beta in 1.36).
* [Kubelet now sets `PodReadyToStartContainers` condition immediately after sandbox creation](https://github.com/kubernetes/kubernetes/pull/134660) rather than after image pull, reducing the time to condition True.
* A [new gRPC service is added to the Kubelet](https://github.com/kubernetes/kubernetes/pull/134627) that provides information about pods running on the node.
* kubectl: [`kubectl describe node`](https://github.com/kubernetes/kubernetes/pull/131744) now lists aggregated ResourceSlices when the `ResourceSlice` API is present, detailing slice name, driver, and pool.
