## Other Merges

* Switch [PLEGOnDemandRelist](https://github.com/kubernetes/kubernetes/pull/137909) default to false for 1.36
* Fixes a 1.35 regression in StatefulSet Parallel pod management by [disabling the MaxUnavailableStatefulSet feature by default](https://github.com/kubernetes/kubernetes/pull/137904).
* Added alpha support for tracking PVC unused status via a [new `Unused` condition on PersistentVolumeClaimStatus](https://github.com/kubernetes/kubernetes/pull/137862).
* Fix bug where users [can't update HPAv2 resources](https://github.com/kubernetes/kubernetes/pull/137856) that use object metrics with averageValue via the v1 HPA API.
* [cri-client helper method NewLogOptions was removed](https://github.com/kubernetes/kubernetes/pull/137827) and LogOptions must be constructed directly.
* Enables Prometheus native histogram support in [kube-proxy](https://github.com/kubernetes/kubernetes/pull/137781),[kubelet](https://github.com/kubernetes/kubernetes/pull/137780), [KCM](https://github.com/kubernetes/kubernetes/pull/137779) when feature gate is enabled.
* The ["Failed to update lease optimistically"](https://github.com/kubernetes/kubernetes/pull/137753) log message may not be shown to users anymore, depending on the log level they have set.
* Add [tiered cgroup v2 memory protection](https://github.com/kubernetes/kubernetes/pull/137719) for `MemoryQoS: memory.min` for Guaranteed pods, `memory.low` for Burstable pods, with node-level metrics and rollback reconciliation (KEP-2570).
* Writes to the [ServiceCIDR main resource](https://github.com/kubernetes/kubernetes/pull/137715) now ignore status field changes in the request, consistent with all other Kubernetes APIs.
* Fixed a bug preventing [Pods sharing ResourceClaims](https://github.com/kubernetes/kubernetes/pull/137647) from being scheduled with GangScheduling.
* Added a deletion protection mechanism for PodGroup objects [KEP-5832](https://github.com/kubernetes/kubernetes/pull/137641).
* Fixed how image names are compared to the values from [`preloadedImagesVerificationAllowlist` in Kubelet's configuration](https://github.com/kubernetes/kubernetes/pull/137629).
* kube-scheduler now updates [PodGroup status with a `PodGroupScheduled`](https://github.com/kubernetes/kubernetes/pull/137611) condition reflecting whether the group was successfully scheduled or is unschedulable.
* [DRA allocation timeouts are now surfaced](https://github.com/kubernetes/kubernetes/pull/137607) as visible scheduling failures and retried with per-pod exponential backoff, instead of silently skipping nodes or incorrectly marking pods unschedulable.
