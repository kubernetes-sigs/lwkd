## Developer News
**Developer News**

Kubernetes has introduced a new [GitHub Actions security policy](https://github.com/kubernetes/community/blob/main/github-management/github-actions-policy.md) requiring all workflows to pin actions using full 40-character commit SHAs instead of mutable references like `latest`, `main`, or tags, to prevent supply chain attacks.
This follows recent security incidents, including the [Trivy GitHub Actions](https://github.com/aquasecurity/trivy/security/advisories/GHSA-69fq-xp46-6x23) vulnerability.
Maintainers must update workflows by April 15, 2026. After this, non-compliant workflows will fail due to enforced policy.
Dependabot is recommended to keep pinned actions up to date.      
Read more: https://groups.google.com/a/kubernetes.io/g/dev/c/p2oS549vY10/m/_FkwoASqAAAJ and If you have questions, please reach out on the [#github-management]( https://kubernetes.slack.com/archives/C01672LSZL0) slack channel.

The [default branch of the kubernetes/community repository](https://groups.google.com/a/kubernetes.io/g/dev/c/p2oS549vY10/m/_FkwoASqAAAJ) has been renamed from `master` to `main`.
Open PRs have been automatically retargeted, and existing /master/ links will continue to work via redirects.
Contributors using local clones will need to switch to the new branch, and fork owners are encouraged to update their default branch to stay aligned.  
If you have questions or concerns, please comment on the [tracking issue](https://github.com/kubernetes/community/issues/6290) or reach out in [#github-management](https://app.slack.com/client/T09NY5SBT/C01672LSZL0) on Slack.
