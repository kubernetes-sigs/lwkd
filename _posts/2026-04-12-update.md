## Developer News

[Kubernetes v1.36.0-rc.0 is live](https://groups.google.com/a/kubernetes.io/g/dev/c/KTEX8AG7l6c), built with Go 1.26.0.

The Steering Committee has [published an updated AI usage policy](https://groups.google.com/a/kubernetes.io/g/dev/c/7Y9016gdFZw) where contributors must disclose AI use in PR descriptions, and AI tools may not be listed as co-authors or co-sign commits.

[CVE-2026-3865](https://groups.google.com/a/kubernetes.io/g/dev/c/awNhgIeMV2c) is a Medium-severity path traversal vulnerability in the CSI Driver for SMB; upgrade to v1.20.1 or later.

[WG AI Integration has been disbanded](https://groups.google.com/a/kubernetes.io/g/dev/c/I9w-3nPm8n4) after its active projects (agent-sandbox, mcp-lifecycle-operator, kube-agentic-networking) moved to their respective SIGs.

Viktória Spišaková is [stepping down from WG Checkpoint-Restore](https://groups.google.com/a/kubernetes.io/g/dev/c/35fNsXIh4qI) with Andrey Velichkevich nominated as her replacement; lazy consensus deadline is April 17 2026.

The [New Contributor Orientation](https://www.kubernetes.dev/docs/orientation/) is next week on [Tuesday April 21](https://www.kubernetes.dev/resources/calendar/).  This week is the first of the new SIG-run format; SIG-CLI is offering this one, so if you wanted to get started contributing to kubectl, join them.

## Release Schedule

**Next Deadline: Kubernetes v1.36.0 Release, April 22**

Kubernetes [v1.36.0-rc.0](https://git.k8s.io/kubernetes/CHANGELOG/CHANGELOG-1.36.md) is now available, built with Go 1.26.0.

Docs Freeze for v1.36 landed last week, and the [release-1.36](https://github.com/kubernetes/kubernetes/tree/release-1.36) branch has been created as we move into the final stages of the release cycle.

Cherry-picks for the April patch releases closed April 10, with the release targeted for April 14.

## Other Merges

* Fixed running of [DRA e2e tests in air-gaped clusters](https://github.com/kubernetes/kubernetes/pull/138318) or with test images in private registries. 
* [The device manager now](https://github.com/kubernetes/kubernetes/pull/138244) restricts iteration to NUMA nodes that actually host devices for the requested resource, reducing the search space to O(2^k) where k is typically 1–2.
* Fix [apiserver startup failure](https://github.com/kubernetes/kubernetes/pull/137677) during upgrade when MultiCIDRServiceAllocator is enabled and the cluster has a large number of namespaces.
