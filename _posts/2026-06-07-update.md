## Steering Committee Meeting

This month's [Steering Committee Meeting](https://docs.google.com/document/d/1qazwMIHGeF3iUh5xMJIJ6PDr-S3bNkT8tNLRkSiOkOU/edit?tab=t.0#heading=h.5gstfx3y3p7i) started out by discussing the 2026 Steering Committee Elections. Based on the feedback from last year's election retro, we are starting the process earlier this year. Antonio Ojea, Benjamin Elder and Sascha Grunert will be stepping down this year and there will be 3 spots to be elected. One feedback from the steering committee was to announce the elections earlier before folks are out for their summer vacations. Keep a lookout for the steering committee elections anouncement soon!

The details of the Maintainer Summit was also discussed. The Maintainer Summit event will happen on the day before the colocated events and will only be happening at KubeCon EU and NA. If you want to have a booth in the project pavillion for a SIG or a subproject, you can request this via SIG ContribEx. The CFP announcement for the Maintainer Summit at KubeCon NA 2026 is not out yet, but is expected to be announced soon.

## Featured PRs
* [138488: DRA Extended Resource: promote to GA in 1.37](https://github.com/kubernetes/kubernetes/pull/138488)
  
 In this pull request, [yliaog](https://github.com/yliaog) promoted Dynamic Resource Allocation (DRA) Extended Resources to GA for the v1.37 milestone to establish a stable interface for managing specialized hardware accelerators. Driven and approved by contributors like [pohly](https://github.com/pohly) , [liggitt](https://github.com/liggitt) , and [bart0sh](https://github.com/bart0sh) , the PR resolves tracking issue [#138376](https://github.com/kubernetes/kubernetes/issues/138376) and aligns with the `wg/device-management` working group.

 * [139466: KEP-961: MaxUnavailable Feature On By Default](https://github.com/kubernetes/kubernetes/pull/139466)

 In this pull request, [soltysh](https://github.com/soltysh) advanced a long-awaited capability by enabling the `maxUnavailable` rolling update strategy configuration by default for [KEP-961](https://github.com/kubernetes/enhancements/issues/961). This change allows for tighter control over application availability and faster rollout tempos during workload updates without relying on manual feature gate activation. 

* [139397: Introduce Feature Gate for IPVS](https://github.com/kubernetes/kubernetes/pull/139397)

[adrianmoisey](https://github.com/adrianmoisey) introduced a brand-new feature gate structurally isolating IPVS (IP Virtual Server) logic within `kube-proxy` for [KEP-5495](https://github.com/kubernetes/enhancements/issues/5495). This lays the crucial architectural groundwork for how the networking subsystem is managed, configured, and potentially decoupled in future releases, affecting both contributors modifying proxy code and users leveraging high-performance IPVS routing modes. Please test that this feature gate functions correctly in your networking setup.

## Other Merges

* Adds the [`EtcdRangeStream` feature gate](https://github.com/kubernetes/kubernetes/pull/136915) which initializes the watch cache via a single server-streaming etcd RangeStream RPC instead of paginated range requests, reducing apiserver memory pressure during bootstrap as part of KEP-5966.
* Adds the [`KubeProxyIPVS` feature gate](https://github.com/kubernetes/kubernetes/pull/139397) as Phase 2 of KEP-5495, gating IPVS mode in kube-proxy on an opt-in basis ahead of full removal in a future release.
* kubeadm: [removed the v1beta3 configuration API and the `PublicKeysECDSA` feature gate](https://github.com/kubernetes/kubernetes/pull/136016); users still on v1beta3 must migrate to v1beta4 before upgrading.
* Fixes a [scheduler bug where gated pods retained `WasFlushedFromUnschedulable` after a flush event](https://github.com/kubernetes/kubernetes/pull/139330), causing them to be repeatedly re-evaluated.
* Fixes a [DRA shared in-flight allocation accounting bug](https://github.com/kubernetes/kubernetes/pull/139418) where error paths left stale entries that could lead to scheduling deadlocks.
* Handles [corrupted subpath mount points during container restart](https://github.com/kubernetes/kubernetes/pull/138856), allowing kubelet to recover automatically instead of failing the restart.
* Fixes a [MemoryQoS bug where BestEffort pods received an incorrect `memory.high` cgroup value](https://github.com/kubernetes/kubernetes/pull/138139), affecting memory-pressure behaviour for those workloads.
* kubelet now [clears stale `memory.high` on containers when MemoryQoS is disabled](https://github.com/kubernetes/kubernetes/pull/139377), preventing residual cgroup configuration when the feature is toggled off.
* Adds a [new Windows node-level e2e test suite](https://github.com/kubernetes/kubernetes/pull/138235) under `test/e2e_node_windows`, closing a long-standing Windows test coverage gap.
