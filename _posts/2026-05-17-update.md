## Featured PRs

### [139010: Reduce numbers of conversions in CEL evaluation pipeline](https://github.com/kubernetes/kubernetes/pull/139010)

Common Expression Language (CEL) admission evaluation pipeline has been optimized to avoid repeatedly converting the same API object during policy evaluation.  The CEL admission plugin backs ValidatingAdmissionPolicy and related features, and converts each `runtime.Object` into a CEL-evaluable representation before a policy expression can run against it. When a request is subject to multiple policies or webhooks, the same object was being converted again for every evaluation, and that conversion became a measurable CPU bottleneck on busy API servers. Under a 200 QPS load test with five ValidatingAdmissionPolicies, the change reduces API server CPU usage from roughly 1.10 cores to 0.93 cores, an approximately 15% reduction, while leaving light-policy workloads functionally unchanged.

The fix introduces a `LazyObject` abstraction wrapping `VersionedObject` and `VersionedOldObject` in `VersionedAttributes`. `LazyObject` caches the CEL `ref.Val` representation on first use and automatically clears it whenever the underlying object is mutated via `Set()`, so the conversion cost is paid at most once per object per request,  and not at all when CEL evaluation is skipped, such as for empty expression groups. Encapsulating the cache this way also resolves a class of desynchronization bugs where an object mutated during mutating admission could leave a stale CEL representation behind. The PR is scoped to SIG API Machinery, and is targeted at the v1.37 milestone.

## Other Merges

* Fixed a bug in [ImageLocality scoring](https://github.com/kubernetes/kubernetes/pull/138951) where image volumes could receive a higher score than equivalent regular container images.
* kubeadm: [fixed a panic in kubeadm PKI key loading](https://github.com/kubernetes/kubernetes/pull/138939) when the private key type and public key type mismatch.
* Fixed a regression where kubelet did not clear [stale cgroup v2 memory.min and memory.low values](https://github.com/kubernetes/kubernetes/pull/138903) when the MemoryQoS feature gate was disabled after being previously enabled.
* kubelet: [the eviction manager's monitoring goroutine](https://github.com/kubernetes/kubernetes/pull/138854) now exits promptly when the kubelet's context is cancelled, fixing a goroutine leak on shutdown.
* Fixed a theoretical issue where [nodes might have been denied access](https://github.com/kubernetes/kubernetes/pull/138792/changes) to synthesized ResourceClaims for pods using extended resources (e.g. nvidia.com/gpu), causing containers to get stuck in ContainerCreating. Not observed in practice.
* Add metric [`apiserver_watch_cache_initialization_duration_seconds` ](https://github.com/kubernetes/kubernetes/pull/138767)recording the duration of the most recent watch cache initialization, labeled by group and resource.
* [The scheduler now avoids redundant preemption attempts during PodGroup scheduling](https://github.com/kubernetes/kubernetes/pull/138710/changes) when terminating victim pods are already present on the nominated nodes.
* kube-controller-manager and kube-scheduler now both expose [`dynamic_resource_allocation_resourceclaim_creates_total`](https://github.com/kubernetes/kubernetes/pull/138542) as metric for number of ResourceClaims created, replacing differently names metrics in each component.
* Added [`net.ipv4.tcp_slow_start_after_idle` and `net.ipv4.tcp_notsent_lowat`](https://github.com/kubernetes/kubernetes/pull/138389) to the allowed safe sysctls list.
* API Go types switched the json tag for inlined TypeMeta fields from [`",inline"` to simply `""`](https://github.com/kubernetes/kubernetes/pull/138260). 
* Removed an [edge case that could allow malformed object deletion](https://github.com/kubernetes/kubernetes/pull/137582) to bypass admission and graceful deletion of well-formed objects.
* Changed the `PatchPodStatus` API in the scheduler framework to accept a [slice of Pod conditions (`[]*v1.PodCondition`)(https://github.com/kubernetes/kubernetes/pull/135160) instead of a single condition (`*v1.PodCondition`). This allows scheduler plugins to update multiple Pod conditions in a single API call, preventing newer calls from overwriting older ones when multiple conditions need to be updated concurrently.
